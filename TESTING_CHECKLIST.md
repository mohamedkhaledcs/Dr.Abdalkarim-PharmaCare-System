# ✅ قائمة تحقق Checklist للاختبار والتشغيل

## 📋 قبل البدء

- [ ] Node.js مثبت (v16+)
- [ ] npm أو yarn متاح
- [ ] حساب Supabase نشط
- [ ] .env.local موجود مع المفاتيح

---

## 🔍 تحقق من الملفات المهمة

```
✅ البحث عن الملفات:
```

- [ ] ملف `package.json` موجود
- [ ] ملف `.env.local` موجود ومعبوء
- [ ] مجلد `src/` موجود كامل
- [ ] ملف `schema.sql` موجود (للبيانات)
- [ ] ملف `data.sql` موجود (للبيانات)

---

## 🛠️ إعداد قاعدة البيانات

### في Supabase Dashboard:

1. [ ] اذهب إلى: https://supabase.com/dashboard
2. [ ] اختر المشروع الصحيح

### تشغيل SQL Scripts:

1. [ ] اذهب إلى SQL Editor
2. [ ] اضغط "New Query"
3. [ ] انسخ كل محتوى `schema.sql` والصقه
4. [ ] اضغط "Run" (يجب أن تظهر رسالة نجاح)
5. [ ] اضغط "New Query" مرة أخرى
6. [ ] انسخ كل محتوى `data.sql` والصقه
7. [ ] اضغط "Run" (يجب أن تظهر رسالة نجاح)

### التحقق:

- [ ] جدول `products` موجود
- [ ] جدول `users` موجود
- [ ] جدول `orders` موجود
- [ ] جدول `categories` موجود

---

## 🎯 فحص RLS Policies

في Supabase:

1. [ ] اذهب إلى Authentication → Policies
2. [ ] تحقق من جدول `products`:
   - [ ] يوجد policy "SELECT (anon)" أم "SELECT (public)"
   - [ ] إذا لم يكن: اضغط "Enable RLS" ثم "New Policy" 
   - [ ] اختر "SELECT" و "public" يقرأ

3. [ ] تحقق من الجداول الأخرى بنفس الطريقة

## ✨ اختبار الميزات

افتح المتصفح وجرّب:

### 1. صفحة Health Check
```
http://localhost:3000/api/health
```
- [ ] ترى JSON بـ `"status": "ok"`
- [ ] ترى `"database": "connected"`
- [ ] ترى عدد المنتجات > 0

### 2. API Products
```
http://localhost:3000/api/products
```
- [ ] ترى مصفوفة JSON من المنتجات
- [ ] كل منتج له: name, price, category, etc.

### 3. الموقع الرئيسي
```
http://localhost:3000
```

#### في الصفحة، شيّك على:

**Hero Section:**
- [ ] يظهر العنوان الكبير
- [ ] زر البحث ظاهر
- [ ] اللون الأخضر واضح

**Navbar:**
- [ ] لوجو صيدلية ظاهر (في الأعلى اليسار)
- [ ] كلمات التصنيفات بالعربية: ألم، مضادات حيوية، إلخ
- [ ] رمز السلة في الأعلى اليمين

**Featured Products:**
- [ ] يعرض 4 منتجات (أو skeletons إذا كان يحمّل)
- [ ] كل منتج يظهر: صورة، اسم، سعر
- [ ] عند hover: الزر يرتفع قليلاً

**Category Filters:**
- [ ] 6 أزرار بألوان مختلفة
- [ ] النصوص بالعربية
- [ ] عند الضغط: يتنقى المعرض

**All Products Grid:**
- [ ] شبكة من المنتجات (2 في الهاتف، 3 في الجهاز المكتبي)
- [ ] عند التحميل: يعرض skeletons مع shimmer effect
- [ ] بعد التحميل: يعرض المنتجات الفعلية

## 🔧 استكشاف الأخطاء

### ❌ لا يظهر "status": "ok"

```
👉 الحل:
1. تحقق من .env.local (اكتب الـ URL والـ Key بشكل صحيح)
2. أعد تشغيل schema.sql ثم data.sql
3. تحقق من RLS policies
4. أعد بدء الخادم (Ctrl+C ثم npm run dev)
```

- [ ] تم التحقق من .env.local
- [ ] تم تشغيل SQL scripts مرة أخرى
- [ ] تم فحص RLS policies

### ❌ لا تظهر المنتجات

```
👉 الحل:
1. افتح http://localhost:3000/api/health
2. شيّك على "totalProducts"
3. إذا كان 0: أعد تشغيل data.sql
4. إذا كان > 0: المشكلة في الـ frontend (refresh)
```

- [ ] تم فحص /api/health
- [ ] تم التحقق من totalProducts
- [ ] تم مسح الذاكرة المؤقتة (Ctrl+Shift+Delete)

### ❌ أخطاء في Console

```
👉 الحل:
1. اضغط F12 لفتح Developer Tools
2. شيّك على tab "Console"
3. انسخ الخطأ
4. ابحث عنه أو أرسله للمساعدة
```

- [ ] تم فتح Developer Tools (F12)
- [ ] تم قراءة الأخطاء من Console
- [ ] تم البحث عن الحلول
